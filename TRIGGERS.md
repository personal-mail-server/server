# TRIGGERS.md

## 문서 목적
이 문서는 프로젝트 전역에서 사용하는 예약된 트리거 명령어와 그 해석 규칙을 정의한다.

AI는 사용자의 입력이 이 문서에 정의된 트리거와 일치할 경우, 일반적인 질의응답이 아니라 사전에 정의된 작업 흐름으로 해석해야 한다.

## 기본 원칙
- 트리거는 프로젝트 전역에서 사용하는 예약된 작업 지시어이다.
- 트리거는 문서, 코드, 설정, 로그 등 특정 영역에 한정되지 않는다.
- AI는 트리거 입력 시 해당 트리거에 정의된 절차를 따른다.
- 문서에 정의되지 않은 단어를 임의의 트리거로 해석하지 않는다.
- 트리거 실행 중 충돌, 해석 불가, 정보 부족이 있는 경우에만 사용자에게 다시 확인한다.

## 트리거 목록

### 동기화
#### 의미
프로젝트 기준 문서와 현재 코드, 설정, 관련 결과물의 차이를 검토하고 필요한 동기화 작업을 수행한다.

#### 동작
- AI는 관련 기준 문서를 전체적으로 확인한다.
- AI는 현재 코드, 문서, 설정 상태와 비교한다.
- 차이는 신규, 수정, 삭제로 구분한다.
- AI는 식별된 차이를 기준으로 필요한 동기화 작업을 수행한다.

#### 예외
- 기준 문서 간 충돌이 있는 경우
- 삭제인지 누락인지 판단할 수 없는 경우
- 문서만으로 결정을 내릴 수 없는 경우

## 확장 규칙
- 새로운 트리거는 이 문서에 추가하여 관리한다.
- 새로운 트리거는 `의미`, `동작`, `예외`를 포함해야 한다.
- 기존 트리거의 동작을 변경할 경우 이 문서를 먼저 수정한다.

## 트리거 해석 원칙
- 한국어를 포함한 원본 트리거 명칭은 엄격하게 해석해야 하며, 예를 들어 `동기화`는 예약된 트리거로 정확히 인식해야 한다.
