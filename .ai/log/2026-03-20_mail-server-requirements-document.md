## 변경 이유
- 예약 트리거 `도큐멘트`에 따라 작성한 기존 메일 서버 MVP 범위가 아직 넓었고, 사용자 요청에 맞춰 테스트용 메일 주소 CRUD와 논리 삭제, 유니크 검증, 자동 생성 API 중심으로 범위를 다시 축소해야 했기 때문

## 변경 내용
- 테스트용 메일 주소 관리 요구사항 문서로 범위를 재작성함
- 목록, 상세, 생성, 수정, 삭제 요구와 논리 삭제 원칙을 명시함
- 생성/수정 시 유니크 검증 요구와 서버 유니크 메일 후보 반환 API 요구를 반영함
- 사용자의 직접 입력 가능 요구를 함께 반영함
- 축소된 범위에 맞춰 AI 컨텍스트 문서를 갱신함
- 실제 문서 재작성 이력을 작업 로그에 반영함

## 영향 범위
- 향후 테스트용 메일 주소 API 설계
- 테스트용 메일 주소 프론트엔드 설계
- 논리 삭제 정책 설계
- 유니크 메일 생성 규칙 설계

## 비고
- 삭제된 주소 재사용 여부, 생성 규칙, 수정 가능 필드 범위 등은 후속 상세 문서에서 확정 필요
