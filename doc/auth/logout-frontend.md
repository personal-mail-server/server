# 로그아웃 프론트엔드 문서

## 문서 목적
이 문서는 로그아웃 기능이 프론트엔드에서 어떤 사용자 동작과 상태 전이를 가져야 하는지 정의한다.

본 문서는 프론트엔드 동작 문서이며, 로그아웃 기능의 핵심 규칙은 `doc/auth/logout.md`, API 계약은 `doc/auth/logout-api.md`를 기준으로 한다.

---

## 기준 문서
- 기능 설계 기준 문서: `doc/auth/logout.md`
- API 기준 문서: `doc/auth/logout-api.md`
- 로그인 프론트엔드 기준 문서: `doc/auth/login-frontend.md`

## 범위
이 문서는 현재 구현된 로그아웃 프론트엔드 동작만 다룬다.

포함 범위:
- 로그아웃 액션 진입점
- 로그아웃 요청 전송 동작
- 성공/실패 상태 표시 방식
- 로그아웃 이후 클라이언트 인증 상태 정리
- 전체 세션 무효화 이후 화면 전이 기준

제외 범위:
- 로그인 화면 자체 구조 변경
- 보호 화면 상세 UI 디자인
- 회원가입 UI
- 비밀번호 재설정 UI
- 관리자 강제 로그아웃 UI

---

## 현재 상태 메모
- 현재 저장소의 프론트엔드는 로그인 입력 화면과 인증 이후 상태 표시 영역을 함께 제공한다.
- 로그아웃 액션은 인증된 상태에서 사용자 메뉴를 통해 접근할 수 있어야 한다.
- 본 문서는 현재 구현된 로그아웃 프론트엔드 동작을 기준으로 유지한다.

---

## 확정된 프론트엔드 조건
- 로그아웃 액션은 인증된 사용자 화면의 사용자 메뉴에 둔다.
- 로그아웃 성공 후 로그인 화면(`/`)으로 이동한다.
- 인증 실패 응답을 받으면 클라이언트 인증 정보를 정리한 뒤 로그인 화면(`/`)으로 이동한다.
- 요청 인증 정보는 `Authorization: Bearer <accessToken>` 형식을 사용한다.

---

## 화면 진입점
- 로그아웃 액션은 인증된 사용자 화면에서 접근 가능해야 한다.
- 로그인 단일 화면에는 로그아웃 버튼이 존재해서는 안 된다.
- 보호 화면의 사용자 메뉴에서 로그아웃 액션을 제공하는 것을 기준으로 한다.

---

## 화면/동작 구조
구성 요소:
- 사용자 메뉴 내부의 로그아웃 액션
- 요청 진행 상태 표시
- 성공 시 로그인 화면(`/`)으로 이동하는 흐름
- 실패 시 오류 상태를 보여주는 메시지 영역

---

## 제출 흐름
1. 사용자가 인증된 화면에서 로그아웃 액션을 누른다.
2. 프론트엔드는 중복 요청을 막기 위해 로그아웃 액션을 일시 비활성화한다.
3. 프론트엔드는 `POST /api/v1/auth/logout` 요청을 전송한다.
4. 요청에는 `Authorization: Bearer <accessToken>` 헤더를 포함한다.
5. 성공 응답을 받으면 프론트엔드는 보유 중인 인증 상태를 모두 제거한다.
6. 프론트엔드가 유지 중인 사용자 상태, 세션 표시, 보호 화면 접근 상태를 비인증 상태로 되돌린다.
7. 사용자를 로그인 화면(`/`)으로 이동시킨다.

---

## 응답 처리 방식

### 성공 응답
- `204 No Content`일 때 성공 상태로 처리한다.
- 성공 메시지를 보여줄 수 있다.
- 클라이언트에 남아 있던 액세스 토큰, 리프레시 토큰, 사용자 인증 상태를 모두 제거해야 한다.
- 이후 보호 화면 접근은 차단되어야 한다.

### 인증 실패 응답
- `401 Unauthorized`일 때 인증 실패로 처리한다.
- 이 경우에도 클라이언트 인증 정보를 정리해야 한다.
- 사용자를 로그인 화면(`/`)으로 이동시켜야 한다.

### 서버 오류 응답
- `500 Internal Server Error`일 때 서버 오류로 처리한다.
- 이 경우 클라이언트는 사용자가 다시 시도할 수 있도록 현재 화면 상태를 유지할 수 있다.
- 단, 서버가 로그아웃 성공을 확정하지 않았으므로 인증 상태를 임의로 성공 처리해서는 안 된다.

### 네트워크 오류
- 네트워크 오류나 예기치 않은 요청 실패 시 프론트엔드 전용 오류 코드를 사용할 수 있다.
- 이 경우 사용자가 재시도 가능하도록 오류 메시지를 보여줘야 한다.

---

## 클라이언트 상태 정리 원칙
- 로그아웃 성공 시 클라이언트 저장소에 남아 있는 인증 관련 값은 모두 제거해야 한다.
- 프론트엔드 메모리 상태, 사용자 표시 상태, 보호 화면 접근 상태도 함께 초기화해야 한다.
- 현재 저장소는 토큰 저장 전략이 미정이므로, 실제 저장 위치는 구현 시점에 결정하되 로그아웃 시 제거 원칙은 유지해야 한다.

---

## 검증 기준
- 인증된 사용자 화면에서 로그아웃 액션이 보여야 한다.
- 로그아웃 액션은 사용자 메뉴 안에 있어야 한다.
- 로그아웃 요청 중에는 중복 제출이 방지되어야 한다.
- 성공 시 비인증 상태로 전환되어야 한다.
- 성공 후 로그인 화면(`/`)으로 이동해야 한다.
- 성공 후 보호 화면 재접근은 재인증 없이 허용되지 않아야 한다.
- 인증 실패 응답 시 클라이언트 인증 정보를 정리한 뒤 로그인 화면(`/`)으로 이동해야 한다.
- 서버 오류 시 성공으로 오인되지 않아야 한다.

- 실제 보호 화면 UI

- 다중 탭 동기화 처리
- 브라우저 자동화 테스트

## 정리
로그아웃 프론트엔드는 인증된 사용자 화면에서 명시적으로 로그아웃을 실행하고, 성공 시 클라이언트 인증 상태를 전부 제거한 뒤 비인증 진입점으로 복귀시키는 동작을 기준으로 한다.

또한 서버가 계정 전체 세션을 무효화하는 모델을 사용하므로, 프론트엔드는 단순 버튼 처리만이 아니라 인증 상태 초기화와 보호 화면 접근 차단까지 함께 수행해야 한다.
