# 테스트용 메일 주소 후보 생성 API 문서

## 문서 목적
이 문서는 테스트용 메일 주소 후보 생성 API의 요청/응답 계약과 동작 규칙을 정의한다.

---

## 기준 문서
- `doc/domain/mail/server-requirements.md`
- `doc/domain/mail/pr-split/02-generate-api.md`
- `doc/domain/mail/test-address-model.md`

---

## 엔드포인트
- 메서드: `POST`
- 경로: `/api/v1/mails/generate`

---

## 인증
- Bearer 액세스 토큰이 필요하다.
- 인증되지 않았거나 세션 버전이 맞지 않으면 실패해야 한다.

---

## 요청 본문
- 없음

---

## 성공 응답
```json
{
  "email": "test-a1b2c3d4e5f6@mail.local"
}
```

## 응답 필드
- `email`: 현재 생성 시점 기준으로 사용 가능한 테스트용 메일 주소 후보

---

## 동작 규칙
- 서버는 현재 저장되지 않은 이메일 후보를 반환해야 한다.
- 반환값은 생성 시점 기준 유니크 후보여야 한다.
- 최종 생성 시점에는 다시 유니크 검증이 필요하다.
- 현재 구현 기본 도메인은 `mail.local` 이다.

---

## 실패 응답

### 401
- 코드: `INVALID_ACCESS_TOKEN`
- 의미: 액세스 토큰이 없거나 유효하지 않거나 세션 버전이 맞지 않음

### 500
- 코드: `INTERNAL_SERVER_ERROR`
- 의미: 후보 생성 또는 충돌 검사 중 내부 오류 발생

---

## 제외 범위
- 후보 이메일 저장
- 테스트용 메일 주소 생성 API
- 프론트엔드 UI

---

## 정리
이 API는 인증된 사용자가 생성 화면에서 바로 사용할 수 있는 테스트용 메일 주소 후보를 반환하기 위한 보호 API이다.
