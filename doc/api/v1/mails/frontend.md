## 화면 구성

### 로그인 전
- 기존 로그인 폼을 그대로 사용한다.

### 로그인 후
- 세션 요약 영역을 유지한다.
- 아래에 다음 3개 영역을 표시한다.
  - 주소 목록 패널
  - 상세 정보 패널
  - 새 주소 생성 패널
- 상세 패널 안에서 수정/삭제 액션을 수행한다.

---

## 사용자 흐름

### 목록 조회
- 로그인 성공 직후 주소 목록을 자동으로 불러온다.
- 사용자는 새로고침 버튼으로 목록을 다시 불러올 수 있다.

### 상세 조회
- 목록에서 주소를 클릭하면 상세 패널에 선택 주소 정보가 표시된다.

### 직접 입력 생성
- 사용자는 이메일을 직접 입력해 생성할 수 있다.
- 생성 성공 시 목록과 상세가 즉시 갱신된다.

### 후보값 생성
- 사용자는 후보 생성 버튼으로 추천 이메일을 받아 입력창에 채울 수 있다.
- 후보값은 그대로 사용하거나 수정 후 생성할 수 있다.

### 수정
- 사용자는 상세 패널에서 현재 주소를 수정할 수 있다.
- 수정 성공 시 목록과 상세 정보가 즉시 갱신된다.
- 중복 이메일 충돌은 상태 패널에 표시한다.

### 삭제
- 사용자는 상세 패널에서 현재 주소를 삭제할 수 있다.
- 삭제 전 브라우저 확인창을 보여 준다.
- 삭제 성공 시 목록에서 제거되고 다른 주소가 있으면 자동 선택한다.

## 상태 표시
- 기존 상태 패널을 그대로 사용한다.
- 목록 로딩, 후보 생성, 생성 성공, 생성 실패, 상세 조회 실패, 수정/삭제 성공 및 실패를 모두 상태 패널에 표시한다.
