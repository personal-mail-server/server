# API 문서 구조 안내

## 목적
이 디렉토리는 API 문서를 URL 루트 기준으로 정리한다.

---

## 구조 원칙
- 루트 경로는 `doc/api/v1/...` 형태로 URL 버전/세그먼트를 반영한다.
- 같은 API 엔드포인트의 백엔드/프론트엔드/설계 문서는 가능한 한 같은 폴더에 둔다.
- 공통 프론트엔드 규칙이 여러 엔드포인트를 포괄할 때는 상위 API 루트에 두고, 각 엔드포인트 폴더에서 참조한다.
- API 상위 요구사항/도메인 모델/구현 계획 문서는 `doc/domain/...`에 두고, API 문서에서 참조만 한다.

---

## 예시
- `doc/api/v1/auth/login/`
  - `design.md`
  - `backend.md`
  - `frontend.md`
- `doc/api/v1/mails/create/`
  - `backend.md`
  - `frontend.md`

---

## 정리
API 문서는 기능명보다 URL 기준 탐색을 우선해 정리하며, API별로 필요한 백엔드/프론트엔드 계약을 한 위치에서 확인할 수 있도록 유지한다.
