# PR 1 - 테스트용 메일 주소 데이터 모델과 저장 기반

## 목표
- 테스트용 메일 주소 관리 기능의 DB 기반과 내부 저장 구조를 먼저 만든다.

## 포함 범위
- 테스트용 메일 주소 테이블 추가
- 소유 사용자 연결 필드 추가
- `deleted_at` 기반 논리 삭제 구조 추가
- 유니크 제약 추가
- 기본 리포지토리/저장 계층 추가

## 제외 범위
- HTTP API
- OpenAPI
- 프론트엔드

## 문서 범위
- 데이터 모델 문서 추가
- 논리 삭제 정책 초안 추가 가능

## 테스트 범위
- 저장 성공
- 유니크 제약 검증
- 논리 삭제 컬럼 기본 동작

## 머지 조건
- 이후 API PR들이 공통으로 사용할 저장 기반이 안정적으로 준비되어야 한다.
