# AGENTS.md

## 기본 규칙
- 모든 응답은 한국어로 작성한다.
- 과대 해석하지 않는다.
- 지시된 작업만 수행한다.

## 작업 원칙
- 명시적으로 요청되지 않은 작업은 진행하지 않는다.
- 추측해서 범위를 넓히지 않는다.
- 불확실한 내용은 사실처럼 단정하지 않는다.
- 불확실한 정보가 있으면 사용자에게 다시 확인한다.

## 역할 분담
- 이 프로젝트에서 사람은 기획 및 설계만 담당한다.
- 코드 작업은 전적으로 AI가 담당한다.
- `project/code` 이하의 구현, 수정, 리팩토링, 정리는 인간이 직접 수행하는 영역이 아니다.
- `project/code` 이하의 작업은 일관되게 AI가 수행한다.
- `.github/workflows`를 포함한 GitHub Actions 관련 작성, 수정, 유지보수도 전적으로 AI가 담당한다.
- AI가 수행한 실제 변경 작업의 로그 작성 및 관리는 전적으로 AI가 담당한다.
- 작업 로그는 `AI_LOGGING.md`의 규칙에 따라 `project/.ai/log` 이하에 기록한다.

## 프로젝트 이해
- 프로젝트의 개요, 목적, 도메인 설명은 `README.md`를 기준으로 이해한다.
- 프로젝트의 기술 스택, 기술 선택 원칙, 운영 기준은 `TECH_STACK.md`를 기준으로 이해한다.
- 예약된 트리거 명령어의 해석과 동작 규칙은 `TRIGGERS.md`를 기준으로 이해한다.

## 작업 로그
- 실제 변경 작업이 발생한 경우 작업 로그를 남긴다.
- 작업 로그는 AI가 검색하고 추적하기 쉬운 형식으로 작성한다.
- 작업 로그 규칙은 `AI_LOGGING.md`를 따른다.

## 문서 기반 작업 원칙
- AI는 개별 지시만 기준으로 작업하지 않는다.
- AI는 작업 시작 시 먼저 `project/.ai/context`의 관련 정리 문서를 우선 조회한다.
- AI는 작업 전 항상 `project/doc` 이하의 전체 문서를 확인한다.
- AI는 기본 작업 해석과 진행에 `project/.ai/context`를 우선 기준으로 사용한다.
- 단, `project/.ai/context`와 원본 문서가 충돌하거나 해석이 불가능한 경우에만 원본 문서를 기준으로 다시 확인한다.
- AI는 `project/doc`의 전체 문서와 현재 코드 상태를 비교하여 동기화가 필요한 변경 사항을 식별한다.
- 동기화 대상은 새로운 내용, 수정된 내용, 삭제된 내용을 모두 포함한다.
- 사용자의 요청이 특정 기능만 가리키더라도, AI는 항상 전체 문서 기준으로 현재 코드와의 차이를 검토한다.
- 구현의 기준은 개별 메시지보다 `project/doc`의 전체 문서 집합이다.
- 이 프로젝트에서는 사람이 코드를 수정하지 않으므로, 문서는 항상 코드보다 우선한다.
- 문서와 코드가 불일치할 경우, 문서가 항상 옳고 코드는 문서에 맞게 수정되어야 한다.
- 기준 문서에 존재하고 코드에 없는 내용은 문서 삭제가 아니라 구현 대상으로 처리한다.
- 기준 문서에 없고 코드에만 존재하는 내용은 제거 대상으로 처리한다.
- `동기화` 수행 시 문서 축소나 문서 재정리보다 코드, 설정, 프론트엔드 반영을 우선한다.
- 단, 문서 간 충돌이나 해석 불가능한 내용이 있으면 사용자에게 다시 확인한다.

## 문서 수정 원칙
- `project/doc`은 사람이 작성하거나 승인한 기준 문서를 관리하는 공간이다.
- AI는 기준 문서를 임의로 기준 변경 용도로 수정하지 않는다.
- AI가 문서에 대한 의견, 수정안, 보완안이 필요하다고 판단한 경우에는 `project/.ai/doc` 이하에 별도 제안 문서로 기록한다.
- `project/.ai/doc`의 문서는 사용자 승인 전까지 기준 문서로 사용하지 않는다.
- 사용자가 작업을 지시했을 때 AI는 `project/doc`의 문서를 우선 기준으로 사용한다.

## AI 정리 문서
- AI는 프로젝트 전체 원본 문서를 더 쉽게 해석하고 작업하기 위해 `project/.ai/context` 이하에 AI 전용 정리 문서를 생성하거나 갱신할 수 있다.
- `project/.ai/context`의 문서는 사람 가독성보다 AI의 검색성, 구조화, 정합성 유지에 유리한 형태를 우선한다.
- `project/.ai/context`의 문서는 기준 문서가 아니며, 원본 문서를 대체하지 않는다.
- 원본 문서가 하나라도 수정되면 관련 AI 정리 문서도 즉시 동기화 대상이 된다.
- 특히 `도큐멘트` 트리거로 기준 문서를 생성하거나 수정한 경우, AI는 관련 `project/.ai/context` 정리 문서를 같은 작업 흐름 안에서 반드시 생성하거나 갱신해야 한다.
- 관련 `project/.ai/context` 문서가 아직 없으면 새로 만들고, 이미 있으면 변경된 원본 문서 기준으로 즉시 갱신한다.
